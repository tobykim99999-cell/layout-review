# 学校规则库目录

真实业务不要直接使用 `default_undergraduate` 演示规则。每个学校、学院、学历层次、模板版本都应该有独立规则库。

## 创建规则库

方式一：手动复制模板。

1. 复制 `school_profile_template.json`。
2. 重命名为目标规则，例如 `xx_university_undergraduate_2026.json`。
3. 修改 `profile_id`、`display_name`、`version`。
4. 在 `source` 中写明学校官方规范、模板文件、发布日期、适用范围。
5. 根据学校规范逐条修改 `rules` 和 `required_sections`。
6. 使用网页下拉框选择该规则库后再上传论文审核。

方式二：网页上传。

1. 打开本地网页。
2. 点击“上传学校论文格式规范或规则库”。
3. 上传 `.json` 结构化规则库，系统会校验并加入下拉框。
4. 上传 `.docx/.pdf/.txt/.md` 官方规范时，系统只保存来源文件并生成草稿，不会自动启用。
5. 将官方规范逐条录入草稿 `rules` 后，把 `is_template` 改为 `false`。

## 规则来源优先级

1. 学校官方毕业论文格式规范。
2. 学校官方 Word 模板。
3. 学院/专业补充要求。
4. 人工确认合格的样本文档。
5. 自研排版系统历史问题数据。

## 注意

规则库是审核标准。智能体和 LLM 都不能凭空决定学校规范。
